# Calisthenics Dashboard Prototype

This repository contains the front-end prototype for a comprehensive **Calisthenics Dashboard** application. This project is designed to be a central hub for calisthenics enthusiasts, providing tools for tracking progress, scheduling workouts, setting goals, and finding motivation.

## Demo:
ethcocoders.github.io/Calisthenics-Prototype/index.html

The prototype showcases a dark-themed, responsive dashboard interface with a rich set of features, each accessible via the main navigation sidebar:

| Feature | Icon | Description |
| :--- | :--- | :--- |
| **Home** | <i class="fas fa-home"></i> | The main dashboard view, providing an overview of key statistics and recent activities. |
| **Workouts** | <i class="fas fa-running"></i> | A section dedicated to listing and managing various calisthenics workout routines. |
| **Calendar** | <i class="fas fa-calendar-alt"></i> | A visual calendar for planning and tracking workout days and rest periods. |
| **Schedule** | <i class="fas fa-clock"></i> | Detailed scheduling for daily and weekly training sessions. |
| **Health** | <i class="fas fa-heartbeat"></i> | A space to monitor and log health metrics relevant to training, such as weight, sleep, and recovery. |
| **Plan** | <i class="fas fa-tasks"></i> | A feature for creating and managing long-term training plans and milestones. |
| **Tournament** | <i class="fas fa-trophy"></i> | A unique section for tracking and participating in calisthenics-related tournaments or challenges. |
| **Goals** | <i class="fas fa-bullseye"></i> | A dedicated area for setting, tracking, and achieving personal fitness goals. |
| **Motivation** | <i class="fas fa-fire"></i> | Content designed to inspire and motivate users throughout their fitness journey. |
| **Videos** | <i class="fas fa-video"></i> | A library for accessing instructional and inspirational calisthenics videos. |

The design is built using pure HTML and CSS, featuring a modern dark theme with a clean, professional layout.

This project is currently a **front-end prototype**. It demonstrates the intended user interface and user experience (UI/UX) of the final application. All navigation and visual elements are in place, but the data displayed is static and the forms/buttons do not connect to a live backend database.
